Querying measurement data

Syntax

Command

 

Query

:MEASure?   <no data/ ALL>
(ALL can be specified only in continuous measurement mode.)

Response

<Measurement data>
(The response format varies depending on the mode.)

ASCII format

<LCR mode>

The AC status is output when the response parameters include any of Z, Y, PHASE, Cs, Cp, D, Ls, Lp, Q, Rs, G, Rp, X, B, S and E, and the DC status is output when the parameters include Rdc.

Binary format

<LCR mode>

The AC status is output when the response parameters include any of Z, Y, PHASE, Cs, Cp, D, Ls, Lp, Q, Rs, G, Rp, X, B, S and E, and the DC status is output when the parameters include Rdc. The binary data is big-endian.

Parameter


Explanation

Command

Query

Returns the status, judgment results, measurement values, and panel number. Set the response content with the :MEASure:VALid command.
When performing normal measurement in LCR mode, you can select the type of parameters to return with the :MEASure:ITEM command.
But, in continuous mode, the ":MEASure:VALid" setting of a panel carried out first is reflected after continuous measurement was finished. So, if you want to use response content unlike the setting of a panel carried out first, please change response content with the :MEASure:VALid command each time.
For details, refer to the :MEASure:ITEM command.
Switching between ASCII mode and binary mode can be set with the :FORMat:DATA command.
During continuous measurement, a message terminator or slash (/) is inserted for each single panel measurement.
When :MEASure? : Message terminator
When :MEASure? ALL : Slash (/)

Example

Command

Query

:MEASure?

Response

ASCII format

<LCR mode>


Binary format

<LCR mode>
